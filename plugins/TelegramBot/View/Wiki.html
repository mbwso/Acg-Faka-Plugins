🚀 30 秒上手

默认 Webhook 模式：零常驻进程，装好填好就能用。
Telegram 主动 POST 消息到 {$webhookUrl}，主程序 PHP-FPM 自动处理。
  1. 在 @BotFather 创建 Bot，复制 Token
  2. 把站点 HTTPS 域名填进「Webhook 域名」配置项（或先在主程序「网站设置」配好 shop_url）
  3. 把 Bot Token 填进配置，保存 → 启用插件
  4. 插件会自动调 setWebhook 完成注册；@你的Bot 发 /start 即可使用

第 1 步：在 @BotFather 创建机器人

  1. Telegram 搜索 @BotFather 并开启对话
  2. 发送 /newbot，按提示输入 Bot 名字、Bot 用户名（必须以 _bot 结尾）
  3. 得到形如 123456789:AAH-aBcD... 的 Token，复制下来
  4. 建议同时给 Bot 设头像、简介：/setuserpic、/setdescription
  5. 把 Bot 隐私模式关掉（这样它能读群消息用于客服）：/setprivacy → 选你的 Bot → Disable

第 2 步：确认你的 HTTPS 域名

Telegram Webhook 强制要求 HTTPS，且要能从公网访问到。常规商城环境一般都满足；如果你的站点还在 HTTP，先用 Let's Encrypt 或宝塔的「SSL」给域名加证书。

在「Webhook 域名」字段填你的站点根域名，不要带尾部斜杠和路径：

正确示例错误示例
https://shop.example.com ❌ http://shop.example.com（非 HTTPS）
https://api.mydomain.com ❌ https://shop.example.com/（带斜杠）
留空（自动取主程序「网站设置 → 站点 URL」） ❌ https://shop.example.com/plugin/...（带路径）

插件会自动把 Webhook URL 拼成：

{$webhookUrl}

第 3 步：填配置、启用插件

后台 → 应用商店 / 插件管理 → TelegramBot → 配置：

字段说明
Bot Token从 @BotFather 拿到的那串
运行模式默认 Webhook（推荐）
Webhook 域名填你的 https://... 站点根；留空则自动用「站点 URL」
其它字段按需填，详见 功能一览

保存配置后 → 点击「启用」。插件做三件事：

  1. 调 getMe 校验 Bot Token
  2. 自动生成一个 32 字符随机 webhook_secret 写回配置（用于校验请求来自 Telegram）
  3. 调 setWebhook 把 {$webhookUrl} 注册到 Telegram

启用成功后，去「控制台」可以看到 Bot 在线状态和 Webhook 注册状态。

第 4 步（可选）：双向客服群

如果你想用「人工客服」功能（用户私聊 Bot 时消息自动转发到你的管理群，客服回复后自动转回用户）：

  1. 新建一个 supergroup（普通群不行）
  2. 群设置 → 编辑 → 打开 Topics 开关
  3. 把你的 Bot 拉进群，提升为管理员，授予「管理 Topic」权限
  4. 用 @getidsbot 之类工具查到群 ID（形如 -1001234567890），填进「客服管理群 ID」
  5. 把自己（或客服员工）的 Telegram ID 填进「管理员 Telegram ID」
  6. 保存配置 → 客户首次发消息时会自动建一个 Topic，标题是 姓名|TG_ID，客服在该 Topic 内回复即可
客服转发用的是 copyMessage，客户看不到「转发自某人」提示，体验自然。
Topic 模式天然支持多客户隔离 —— 一个 Topic 一位客户，不会串台。

备用方案：Long Polling 模式 无 HTTPS 时用

如果你的站点确实无法上 HTTPS（罕见），切换运行模式为 polling：

方式 1：CLI + supervisord（推荐）

[program:acgshop_telegrambot]
directory=/path/to/Acg-Faka-Local
command=php app/Plugin/TelegramBot/bot.php
autostart=true
autorestart=true
stderr_logfile=/var/log/tgbot.err.log
stdout_logfile=/var/log/tgbot.out.log
user=www-data

方式 2：HTTP cron 定时拉取

  1. 在配置里填 Cron Token（自己起一串随机字符）
  2. 加 cron 任务：
* * * * * curl -s "https://你的站点/plugin/TelegramBot/cli/run?token=你的cron_token" > /dev/null

📦 功能一览

功能说明
🛍 商品浏览分类 → 商品 → 多规格 SKU/race → 自助选号 → 联系方式 → 数量 → 支付方式
💳 聚合支付直接调主程序 OrderService::trade()，复用所有已启用的卡网支付通道（USDT/微信/支付宝/其他）
🔗 账号互通Bot 内绑定/注册 → 网页端一键登录；订单/余额/推广关系实时同步
💬 双向客服Topic 模式，参考 MiHaKun/Telegram-interactive-bot，copyMessage 转发 + reply 链双向映射
🤝 推广分销复用主程序 user.pid，订单付款时主程序原生 Bill::create 自动结算返利
🔔 支付推送客户付款成功 → 自动推送卡密发货内容到 Bot；管理群同步收到新订单通知

🔗 账号互通 / 一键登录

Bot 端 → 网页端：

  1. 已绑定账号的用户，在 Bot 主菜单点「🌐 一键登录网页端」
  2. Bot 生成 10 分钟有效的一次性 SSO token
  3. 用户点击链接 → 自动设置登录 cookie → 跳转「会员中心」

网页端 → Bot 端：

  1. 用户在 Bot 内通过 🫂 绑定账号 输入网页端用户名 + 密码，或 👤 注册账号 直接在 Bot 创建主程序账号
  2. 绑定后，TG 用户 ID 与商城用户 ID 永久关联，下单时自动作为已登录用户处理

🤝 推广分销

本插件不另起一套返利体系，完全复用主程序的「上下级 + 订单分成」逻辑：

  • 已绑定的用户在 Bot 内点击「🤝 推广分销」可看到：下级人数、累计返利、专属推广链接
  • 推广链接形如 https://你的站点?promotion_from=用户ID，分享出去后，新用户从该链接进站注册时会被记 user.pid
  • 下级订单付款成功时，主程序在 OrderService::orderSuccess() 里自动 Bill::create($from, divide_amount, ADD) 入账
  • 本插件 hook USER_API_ORDER_PAY_AFTER，给上级用户的 Bot 发送「返利到账」推送

🛠 管理员指令（在管理群对应 Topic 内执行）

命令作用
/ban封禁当前 Topic 对应的用户，禁止其发送消息
/unban解禁
/clear删除当前 Topic 所有消息映射 + 删除该 Topic（用户下次说话会重建）
/info查看当前 Topic 对应用户的 TG ID / 绑定状态等
指令需要发起者的 TG ID 在「管理员 Telegram ID」白名单内才生效。

❓ 常见问题诊断

启用插件时报错「Telegram Webhook 仅支持 HTTPS」

你填的 webhook_domain 或主程序的「站点 URL」是 http:// 开头。
  1. 给域名上 HTTPS 证书
  2. 或临时切到 polling 模式

启用时报「setWebhook 拒绝」

可能原因：
  • 填的域名拼写错误 / DNS 没解析到你服务器
  • HTTPS 证书无效（自签名、过期、域名不匹配 → Telegram 拒绝）
  • 站点 nginx/防火墙拦了 Telegram IP（可在日志里 grep User-Agent 是否有 TelegramBot）
  • 站点根本无法从公网访问
诊断：用 curl https://你的站点/plugin/TelegramBot/webhook/info 看返回。

Bot 收到消息但不回复

  • 查看 app/Plugin/TelegramBot/runtime.log 有无报错
  • 「控制台」页面看 Webhook 状态：last_error_message 是否非空、pending_update_count 是否堆积
  • 测试 webhook 路径是否被 nginx 路由屏蔽：访问 /plugin/TelegramBot/webhook/info 应返回 JSON

双向客服 Topic 没自动创建

  • 群必须是 supergroup（不是普通 group）
  • 群必须开启 Topics 功能
  • Bot 必须是群管理员
  • Bot 必须拥有「管理 Topic」权限（创建管理员时手动勾选）

商品 SKU 选项名字过长 / 中文超多

Telegram 限制 callback_data ≤ 64 字节。本插件用 base64 编码 SKU 参数，4-5 个中文字符以内安全。请把商品 SKU/race 名称改短。

换了 Bot Token / Webhook 域名后

修改配置 → 保存。插件 SAVE_CONFIG 生命周期会自动重新 setWebhook。如果还有问题，停用 然后 再启用一次即可。

如何重置一切

后台停用插件 → 卸载 → 重装 → 重新启用。所有 5 张数据表会保留（除非你手动 drop）。
