🚀 Webhook 模式 — 零部署

当前是 Webhook 模式，无需任何常驻进程。Telegram 主动 POST 消息到下面这个 URL，PHP-FPM 会处理：

{$webhookInfo.url|default:"（未注册，请检查配置）"}

填好 Bot Token + Webhook 域名 + 启用插件，搞定。

详细文档请看 📖 Wiki。

#{else}
🚀 Polling 模式 — 需要常驻进程

方式 1：Linux supervisord（推荐 7x24）

[program:acgshop_telegrambot]
directory=/path/to/Acg-Faka-Local
command=php app/Plugin/TelegramBot/bot.php
autostart=true
autorestart=true

方式 2：HTTP cron（每分钟拉一次）

① 配置「Cron Token = 一串随机字符」

② 加 cron：

* * * * * curl -s "{$webhookInfo.url|default:"https://你的域名"}/plugin/TelegramBot/cli/run?token=你设置的token" > /dev/null

💡 改回 Webhook 模式可省去这一步。

📋 部署清单
  1. 在 @BotFather 创建 Bot，拿到 Token 填进配置
  2. 「Webhook 域名」填你的 HTTPS 站点根（必须公网可访问的 HTTPS）
  3. 点 启用 插件，自动注册 Webhook
  4. （可选）双向客服：建 supergroup → 开 Topics → 把 Bot 设为管理员授「管理 Topic」权限 → 群 ID 填进配置
  5. Telegram 搜你的 Bot，发 /start 体验

需要更多细节请看 📖 完整使用文档。
